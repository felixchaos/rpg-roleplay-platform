# AUP 滥用举报处理运维手册

**适用范围**: Stellatrix 平台运营团队  
**依据**: Stellatrix 可接受使用政策（AUP）、CODE_COMPLIANCE_CHECKLIST AUP-01..03  
**更新日期**: 2026-05-31

---

## 1. 滥用举报接收渠道

| 渠道 | 说明 |
|------|------|
| 平台内 `/report` 端点 | 用户选择举报类型，自动写入 `abuse_reports`（TODO：v37 确认表名）|
| `abuse@stellatrix.icu` | 外部举报邮件 |
| DMCA 通知 | 转 dmca.md 流程 |
| CSAM 举报 | 转 csam.md 流程 |

---

## 2. 滥用举报分类

| 分类 | 描述 | 处理时限 | 优先级 |
|------|------|----------|--------|
| `csam` | 儿童性剥削内容 | 即时（<1h） | P0 — 转 csam.md |
| `illegal_content` | 其他违法内容（恐吓、仇恨煽动等） | 4h | P1 |
| `dmca` | 版权侵权 | 24h | P1 — 转 dmca.md |
| `spam` | 垃圾内容/广告刷屏 | 48h | P2 |
| `harassment` | 骚扰、人身攻击 | 24h | P2 |
| `impersonation` | 冒充他人 | 24h | P2 |
| `other` | 其他违反 AUP 的行为 | 72h | P3 |

收到举报后，**先分类再按时限处理**；同一举报涉及多类，取最高优先级。

---

## 3. 处理步骤

### 3.1 举报分拣（所有类型通用）

1. 在 Admin UI **→ AUP Actions 标签** 查看待处理举报队列。
2. 评估举报是否完整（举报人 ID、被举报内容 URL、举报原因）。
3. 分配给对应处理人，标记优先级。

### 3.2 内容评估

- 查看被举报内容（仅元数据或摘要，对敏感内容限制直接浏览）。
- 对照 AUP 判断是否违规。
- 记录决定理由（必须书面记录，便于申诉审查）。

### 3.3 处置操作

| 处置 | Admin UI 操作 | API |
|------|--------------|-----|
| 仅隐藏内容（不处理账户） | 内容管理 → 隐藏 | 按内容类型 SQL |
| 临时暂停账户 | AUP Actions → 暂停 | `POST /api/admin/users/{id}/suspend` |
| 永久终止账户 | AUP Actions → 终止 | `POST /api/admin/users/{id}/terminate` |
| 驳回举报（无违规） | 标记 unfounded | 更新举报状态 |

### 3.4 通知被举报用户

- 执行暂停/终止后，通过账户邮箱发送通知（模板见附录 A）。
- 告知：被采取措施的具体内容（不提供举报人信息）、原因摘要、申诉途径（见 appeal.md）。
- **暂停**：同时注明暂停到期时间（如有限期）。

---

## 4. 处理时限总结

| 优先级 | 时限 | 超时上报路径 |
|--------|------|-------------|
| P0（CSAM） | <1 小时 | 立即升级至法务负责人 |
| P1（违法/DMCA） | 4-24 小时 | 超时上报运营负责人 |
| P2（骚扰/冒充） | 24 小时 | 同 P1 |
| P3（其他 AUP） | 72 小时 | 超时记录并说明原因 |

---

## 5. 升级路径

| 触发条件 | 升级对象 |
|----------|---------|
| CSAM 疑似内容 | 法务负责人（即时） |
| 涉及执法机构协作 | 法务负责人 + CEO |
| 同一用户被多次举报（≥3 次同类） | 运营负责人 |
| 涉及管理员账户的举报 | CEO 直接处理 |
| 举报人为政府机构或律师事务所 | 法务负责人 |

---

## 6. 记录保留

所有举报、处置决定和通知记录保留 2 年。涉及法律事务（CSAM/违法内容）按对应手册保留更长时间。
