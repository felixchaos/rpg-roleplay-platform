# 新游戏向导

新游戏向导帮你在开始游戏前完成关键的初始设置。创建存档后系统自动进入向导，共 5 步，全部设置好后才正式进入游戏。

## 向导流程

向导顶部会显示当前步骤进度条，可以点「上一步」随时回头修改。

### 第 1 步：剧本与起始世界线

选择本次游戏从哪条**世界线**开始。世界线是剧本预设的叙事起点（如"默认初始线"或"分歧线 B"），决定了 GM 最初的叙事环境和已发生的历史背景。

如果剧本只有一条世界线，此步骤默认选中，直接下一步即可。

<!-- TODO: 截图 — 世界线选择下拉 -->

### 第 2 步：角色

选择你要扮演的 PC（玩家角色）卡、persona（人格/扮演风格）以及角色的起始状态。这一步走既有角色卡流程，默认设置通常可以直接跳过。

### 第 3 步：元知识

配置你的角色**对世界了解多少**：

- **foreknowledge_mode（前知模式）**：角色是否拥有超出剧情本身的"元知识"（如你读过原著、知道剧情走向）。可选不同程度，影响 GM 如何处理角色的知情程度。
- **npc_awareness（NPC 意识）**：NPC 是否会对玩家的"元知识行为"做出反应（如玩家提前知道某人会叛变）。

### 第 4 步：引导与防剧透

- **steering_strength（引导强度）**：GM 在剧情推进时主动引导的力度。高引导度下 GM 更主动带节奏；低引导度下 GM 更多跟随你的行动。
- **spoiler_guard（防剧透保护）**：开启后 GM 不会主动提示你尚未触达的关键情节，适合想自然探索剧情的玩家。

### 第 5 步：确认

显示你在前四步设置的所有参数，请核对后点「**开始游戏**」。

> 建档后，世界线和角色身份相关的设定**锁死**，无法在游戏中途更改。其余设置（如引导强度、防剧透）游戏中可随时在设置页调整。

## MCP 是什么

MCP（Model Context Protocol）是一种让 GM（AI）连接外部工具的协议。配置 MCP 服务器后，GM 在游戏中就能调用该服务器提供的工具——例如：

- 访问本地文件系统读取玩家提供的素材
- 查询外部数据库或 API
- 调用自定义脚本执行计算

MCP 是**可选的高级功能**，普通玩家无需配置。如果你或服务器管理员配置了 MCP 服务器，游戏内通过**附件菜单**（`+` 按钮）的「MCP 工具」选项，就可以在本轮对话里挂载并使用对应工具。

在平台「扩展 → MCP」页面可以查看和管理已配置的 MCP 服务器。

## Skills 是什么

Skill 包是**规则裁定扩展包**，让 GM 按照特定规则集进行裁定。例如：

- **骰子裁定包**：GM 在关键时刻掷骰（D20、D6 等），根据点数决定成败
- **5E 规则集**：按照 D&D 5th Edition 规则进行属性检定和战斗裁定
- **自定义规则包**：服务器管理员可上传任何规则集

Skill 包通过在游戏内**附件菜单**（`+` 按钮）的「Skill 包」选项挂载，也可以在游戏设置里设为默认启用。在平台「扩展 → Skill」页面可以导入和管理本地部署的 Skill 包。

> 注意：Skill 包是服务器本地部署的，需要管理员或拥有上传权限的用户通过平台后台导入后，普通用户才能使用。

## 小提示

- 向导的字段定义来自后端剧本的 `settings.SETTINGS_SCHEMA`，不同剧本可能提供不同的设置项
- 如果向导加载失败（显示"加载向导…"一直不变），请检查网络连接或联系管理员
- 建档后随时可以在存档的「设置」页查看当前设置（锁死的项目显示为灰色不可编辑）
