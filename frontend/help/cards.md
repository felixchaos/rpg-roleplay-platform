# 卡牌系统

## 这页能干嘛

「卡牌系统」管理游戏里两类角色卡：**用户角色卡**（你扮演的 PC）和 **NPC 角色卡**（从剧本提取的配角）。角色卡是 GM 理解角色设定的核心来源，也可以从 SillyTavern/Chub 导入第三方卡包。

入口：平台顶栏「**角色卡**」，分为「用户角色卡」和「NPC 角色卡」两个子页。

## 关键概念

### 用户角色卡（PC 卡）vs NPC 角色卡

| | 用户角色卡 | NPC 角色卡 |
|---|---|---|
| 代表谁 | 你在游戏里扮演的角色 | 剧本里的配角（由 LLM 提取或手动创建） |
| 归属 | 跨剧本共享，绑定到你的账户 | 绑定到某本具体剧本 |
| 导出 | 可导出为 SillyTavern PNG / JSON | 可「转为用户角色卡」后再导出 |
| 可见范围 | 私有 / 公开 | 剧本内 / 私有 / 公开 |
| 删除方式 | 在卡牌页直接删除 | 在剧本管理页面删除 |

### 角色卡字段说明

无论 PC 还是 NPC，编辑表单包含以下字段：

| 字段 | 说明 |
|---|---|
| **姓名** | 必填，GM 用此识别角色 |
| **全名** | 欧美式完整姓名，与姓名不同时才需要填 |
| **身份 / 职位** | 角色的社会身份，如「南陵巡检」「灯塔守人」 |
| **别名** | 逗号分隔，GM 据此识别同一角色的多种称呼 |
| **标签** | 逗号分隔，用于搜索和筛选 |
| **前史 / 背景** | 角色登场前的来历，GM 在需要时注入上下文 |
| **外貌** | 外观描述 |
| **性格 / 设定** | 核心性格特征 |
| **语气 / 说话风格** | GM 模仿口吻的参考 |
| **当前状态** | 开局时的处境或心境 |
| **关键秘密** | 仅 GM 可见，不会直接暴露给其他角色 |
| **示例对话** | 每行一句，帮助 GM 模仿语气 |

### 注入参数

控制该卡在 GM 上下文窗口里的行为：

- **重要度**（0–100）：越高越优先被检索注入，主角/核心 NPC 建议 80+
- **首次揭示章节**（仅 NPC）：章节闸——在这章之前 GM 不会主动引用该 NPC
- **Token 预算**：注入时占用的最大 token 数，默认 450
- **优先级**：同等重要度时的排序依据，默认 100
- **可见范围**：私有（仅自己）/ 剧本内（随剧本分享）/ 公开（所有人可见）

## 常见任务

### 新建用户角色卡

「用户角色卡」页 → 右上角「**新增角色卡**」→ 填写姓名（必填）和其他设定 → 右侧「**创建**」。

### 导入 SillyTavern 角色卡（酒馆卡）

支持 `.png`（PNG 内嵌元数据格式）、`.json` 和 `.webp`，单次最多 8 个文件。

**文件上传方式：**
1. 「用户角色卡」页 → 右上角「**导入酒馆卡**」。
2. 把文件拖到虚线框，或点击框体选择文件。
3. 系统自动解析并显示预览（名称、格式、简介、开场白、标签）。
4. 确认无误后点「**导入**」。

**粘贴 JSON 方式：**
在导入弹窗切到「粘贴 JSON」，直接粘贴角色卡 JSON 文本，点「**解析并预览**」确认后导入。

> 支持 SillyTavern V2 规范（`spec: "chara_card_v2"`）和 V1 旧格式，字段自动映射。

### 导出 SillyTavern PNG 卡

选中一张用户角色卡 → 详情面板右上角「**导出**」（或行操作下拉「导出 PNG（带卡数据）」），浏览器会下载一个内嵌角色卡元数据的 PNG 文件，可以直接导入 SillyTavern/Chub 等工具。

也可以选「导出 SillyTavern JSON」获取纯文本格式。

### 编辑角色卡

点击列表中的角色卡行 → 下方详情面板展开。切到「**角色设置**」标签，修改字段后点「**保存**」。

详情面板还包含：
- **角色信息**：总览字段（类型、来源、重要度等）
- **设定**：只读美化展示

### NPC 卡转为用户角色卡

「NPC 角色卡」页 → 找到目标 NPC → 右侧下拉「**更多 → 转为用户角色卡**」。系统会把 NPC 的设定字段复制到用户角色卡库，添加标签「源自 NPC」。

### 跨剧本复用

用户角色卡是账户级别的，新开任何剧本时都能在「开始新游戏向导」的「选择角色」步骤里选取。NPC 角色卡只属于当前剧本，跨剧本使用需要先「转为用户角色卡」。

### 置顶角色卡

编辑卡片时，可以在筛选栏切到「**置顶**」查看已置顶的卡。置顶状态通过后端 `pinned` 字段控制，可在编辑表单修改。

## 常见问题

**NPC 角色卡页面是空的？**
说明当前账户下的剧本还没有 NPC 角色卡。先在「剧本管理」里导入剧本，运行 KB 提取后，NPC 角色卡会自动生成并出现在这里。

**导入 PNG 卡看不到设定内容？**
部分 PNG 卡的元数据格式不标准。可以尝试「粘贴 JSON」方式，把卡对应的 JSON 文件粘贴进去。

**角色卡的 Token 预算设多少合适？**
主角建议 600–800，重要 NPC 建议 400–600，次要 NPC 建议 200–400。总注入量不超过模型上下文窗口的 1/4 为宜。

**在 NPC 卡页面点删除提示「去剧本管理页面删除」？**
这是设计限制，NPC 卡的删除入口在「剧本管理 → 选中剧本 → NPC 角色卡标签」里单独操作。
