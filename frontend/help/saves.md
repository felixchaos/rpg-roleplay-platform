# 存档管理

## 这页能干嘛

「存档管理」是你查看、创建、切换游戏存档的地方。每个存档绑定一本剧本和一张角色卡，记录了你的全部游戏历史（每一轮对话都是一个分支节点，类似 git commit）。

入口：平台顶栏「**存档**」→「存档目录」。

## 关键概念

### 自动存档 vs 手动存档

- **自动存档**：每完成一回合对话，系统自动在当前分支末尾生成一个新节点。你不需要手动操作，数据不会丢失。
- **手动存档**：目前通过「分支」机制实现——你可以在任意历史节点「继续」，系统会从该节点新建一条分叉分支，原来的分支完整保留。

### 分支（Branch）

存档的历史记录像 git 一样组织为有向树：

```
开场节点
  └─ 登船后向船工打听
       ├─ 申时落岸（主线）
       │     └─ 码头听闻浮尸三具 ← HEAD（当前）
       └─ 选择借宿渔家旅店（分叉点）
             └─ 旅店遇沈知微
```

- 从**末尾节点**继续 → 沿原分支继续，不产生新分叉。
- 从**中段历史节点**继续 → 自动创建新分叉，原分支保留。

「分支」标签页和「分支图」页展示完整的 swimlane 分支图（VSCode Git Graph 风格），可以可视化看到所有分叉。

### 当前存档

存档列表里标记「在玩」（绿色徽章）的是当前激活存档，进入游戏时默认加载这条。可以点「设为当前」切换活跃存档。

## 常见任务

### 开始新游戏

点右上角「**开始新游戏**」，进入 4 步向导：

1. **存档名称 + 剧本**：填写存档名、选择已导入的剧本。
2. **角色卡**：选择你的 PC 角色卡（或新建一张）。
3. **出生点**：按剧情阶段选择从哪个故事节点开局（初期 / 中期 / 终局等），每个阶段有多个具体锚点可选。
4. **初始身份**：AI 推荐 + 自定义，确定你在故事里的身份定位。

### 继续游戏

- 直接点顶栏右上角「**进入当前游戏**」，跳到当前存档的末尾节点。
- 或在存档列表选中某条存档 → 详情面板「**继续游戏**」。
- 想从历史某节点开始：进入「**分支图**」→ 选中节点 → 「激活」切换到该节点 → 再进入游戏。

### 从历史节点创建分叉

1. 在存档详情面板切到「**分支**」标签，看到节点列表。
2. 点「**打开完整分支树**」进入分支图全页。
3. 在分支图里点选你想回到的历史节点 → 「继续」。
4. 系统会将 runtime 切换到该节点状态，进入游戏后产生的新内容自动成为一条新分叉，原分支不受影响。

> 提示：如果选中的节点不是当前分支末尾，弹窗底部会显示橙色警告「选中 #XX 不是当前分支末端，进入后会新建分叉」，确认无误再继续。

### 导出存档

选中存档 → 详情面板右上角「**导出**」，浏览器下载 `.zip` 格式的存档包，包含：

- 全部分支节点的对话记录
- 当前 runtime 状态快照
- 绑定的角色卡信息

导出用途：备份、迁移到自托管实例、离线归档。

### 导入存档

右上角「**导入存档**」→ 选择 `.zip` / `.json` / `.tar.gz` 文件，系统解析后在存档列表里新增一条记录。

> 导入的存档不会自动激活，需要手动「设为当前」。

### 重命名存档

选中存档 → 详情面板右上角「**重命名**」→ 输入新名称 → 「保存」。

### 删除存档

选中存档 → 详情面板右上角「**删除**」→ 确认弹窗。

> 注意：删除操作不可撤销（但底层 commit 文件在服务器上仍有保留期，如需恢复联系管理员）。

## 跨设备使用

| 场景 | 说明 |
|---|---|
| **SaaS 云端** | 存档实时存到服务器，任意设备登录账户即可访问。 |
| **自托管本地** | 数据在你自己的服务器上，不经过第三方。跨设备需手动「导出 → 导入」或配置共享存储。 |

## 常见问题

**存档列表是空的？**
点右上角「**开始新游戏**」先创建一条存档。需要先在「剧本管理」里导入至少一本剧本。

**进游戏后发现剧情从错误的节点开始？**
退出游戏回到存档管理 → 打开「分支图」→ 找到正确节点 → 「激活」切换 → 再次进入游戏。

**分支图怎么区分主线和分叉？**
每列（swimlane）代表一条分支轨道，圆圈是 commit 节点，圆环（空心圆圈）代表当前 HEAD/active 节点，虚线代表已删除的子树。

**存档设置（游戏设置）里有哪些选项？**
选中存档 → 详情面板「**设置**」标签 → 可修改元知识、引导风格、防剧透等参数。注意：部分字段（如起始世界线、初始身份）为「建档锁死项」，开局后不能修改，修改请求会被后端拒绝并提示。

**删掉的分支还能恢复吗？**
删除分支子树后，界面上不再显示，但服务器 commit 文件在保留期内可由管理员恢复。普通用户无自助恢复入口。
