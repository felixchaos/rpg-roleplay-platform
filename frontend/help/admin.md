# 管理后台

管理后台只对角色为「管理员」的账号可见，入口在平台左侧导航栏底部。共 8 个子页面，覆盖用户、用量、安全、运维等维度。

## 用户管理

<!-- TODO: 截图 — 用户列表页面 -->

**能做什么：**

- 查看所有用户的用户名、显示名、角色、状态、最后登录时间
- 查看每用户 30 天 Token 消耗和活跃 Session 数
- 按用户名 / 显示名搜索，按角色（管理员 / 普通用户）和状态（活跃 / 已停用）过滤

**可执行操作：**

| 操作 | 说明 |
|---|---|
| **停用** | 该用户无法继续登录，但数据保留。可随时恢复。 |
| **恢复** | 重新允许已停用的用户登录 |
| **强制下线** | 使该用户所有 Session 立即失效，需重新登录 |
| **升为管理员** | 授予系统管理权限 |
| **降为普通用户** | 撤销管理权限 |

> 所有危险操作（停用 / 下线 / 角色变更）都会弹出确认对话框，且无法对自己执行停用 / 角色变更。

## 全局用量

<!-- TODO: 截图 — 全局用量页，含按用户排行榜 -->

查看整个平台的 Token 消耗与 API 成本，时间范围可选（7 天 / 14 天 / 30 天 / 90 天）。

- **总览卡片**：总请求数、总 Token（入 + 出）、总成本（USD）
- **按用户排行**：谁用得最多？列出排名、Token 量、成本和占比百分比（带进度条）
- **按 API 分布**：各 AI 提供商各用多少
- **按天趋势**：日粒度的 Token 柱状图，直观看到用量峰值

## 审计日志

记录所有管理员操作与安全事件，每次操作自动入库，不可篡改。

**字段说明：**

| 字段 | 内容 |
|---|---|
| 时间 | 操作发生的时间戳 |
| 操作者 | 执行操作的管理员账号 |
| 操作类型 | 如 `user.deactivate` / `config.save` / `invite.create` 等 |
| 目标 | 被操作的资源（用户名 / 配置项等） |
| 详情 | 可展开查看原始 JSON，包含变更前后的具体值 |
| IP | 操作来源 IP |

可按操作类别过滤（全部 / user.* / config.* / maintenance.* / invite.*）。

## 系统健康

每 30 秒自动刷新，也可手动刷新，查看后端各组件的实时状态：

- **数据库**：连接是否正常、查询延迟（ms）
- **内存**：进程 RSS 内存占用（MB）
- **磁盘**：已用空间百分比（超过 90% 会变警告色）
- **进程**：PID 和已运行时长（分钟）

---

## 系统日志

查看后端运行时日志，支持按行数过滤（50 / 100 / 200 / 500 行）和按级别过滤（ERROR / WARN / INFO）。ERROR 行红色高亮，WARN 行橙色高亮，方便快速定位问题。可点「下载」导出为 `.log` 文件。

## 注册配置 + 邀请码

<!-- TODO: 截图 — 注册配置 + 邀请码管理页面 -->

### 注册模式

三种模式一键切换：

| 模式 | 说明 |
|---|---|
| **开放注册** | 任何人都可以直接注册账号 |
| **仅邀请** | 只有持有有效邀请码的人才能注册 |
| **关闭注册** | 完全关闭新用户注册入口 |

还可以分别开关**邮箱验证**和**自动审批**。

> **注意：** 邀请码 + 邮箱验证的完整注册闸功能（即：注册时必须填邀请码并验证邮箱）正在改造中，**即将上线**。届时"仅邀请"模式将与邮箱验证联动，进一步收紧注册闸口。

### 邀请码生成与分发

1. 点击「**生成邀请码**」按钮
2. 填写：
   - **数量**（1-10 个）
   - **过期天数**（7 / 14 / 30 / 90 / 180 / 365 天）
   - **备注**（可选，记录用途如"内测用户 A 批"）
3. 点「生成」后，邀请码出现在下方列表

将邀请码发给受邀用户，让他们在注册页面填入即可。列表中会显示每个码的状态（可用 / 已使用 @用户名 / 已过期）。未被使用的码可随时删除。

## 安全配置

调整以下安全参数（保存后部分配置需重启服务生效）：

- **速率限制**：每 IP 最大请求数、每用户最大请求数、时间窗口（分钟）
- **密码策略**：最小长度、是否要求数字
- **Session 策略**：Session 超时天数
- **登录锁定策略**：连续失败次数阈值、锁定时长（分钟）
- **IP 黑名单**：每行一个 IP 或 CIDR 段（如 `192.168.1.0/24`）

## 维护模式 + 服务重启

### 维护模式

开启后所有用户访问时会看到维护公告。可在此填写公告正文（支持多行），描述维护内容和预计恢复时间。维护完成后在此关闭即可。

### 服务重启

向后端发送优雅重载（graceful reload）信号。重启期间约 5-15 秒内新请求可能失败，建议先开启维护模式通知用户，再执行重启。
